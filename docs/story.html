01 · the idea

you can't make the model bigger.
so make the system smarter.

A 4B model on a 4GB GPU is weak, and no prompt fixes that. This project takes the other road: memory as plain markdown files, retrieval as a decision policy, quality decisions moved out of the model and into code — and four mechanisms that improve with use, without ever touching a weight.

model does

judgment & content: which memories matter, what the slide outline says, what a method's next step is. Small, forgiving structures — never numbers, never file formats.

code does

everything measurable: scoring bands, language detection, tool routing, rendering, validation, rollback. If it can be deterministic, it is.

02 · one question's journey

retrieval is a policy, not a lookup

Every request is decomposed before it's answered. Rules decide what rules can decide (milliseconds, no model call); the model only fills small schemas. Watch the pulse:

fast path (rules + memory) deliverable path (plan → tools → render) learning feedback (desire paths)
Three-way abstention: if nothing relevant clears the gate, the system retrieves nothing — general knowledge gets answered and tagged, personal gaps get an honest "that's not in my memory". No fabricated facts about you.

04 · the guardian

self-improvement without a gate
is self-destruction on a delay

Microsoft's SkillOpt team measured it[1]: an ungated self-evolving agent adopted one plausible-but-wrong rule and collapsed. The gated twin rejected every bad edit and never lost a point. The chart below reproduces their published ablation numbers.

So we built the gate in. Every self-modification — a tuned threshold, a batch of memory repairs, an evolved method — is measured on held-out tasks and rolled back if the score drops. Live test: a harmful ranking edit took hit@1 from 0.583 to 0.333 → REJECTED, restored to 0.583.
05 · the newest loop

abilities that evolve — AlphaEvolve, pocket edition

DeepMind's AlphaEvolve beat a 56-year-old matrix-multiplication record with one loop[2]: generate variants freely, score them ruthlessly, keep diversity. The same loop now runs here, locally, over method documents — with a deterministic evaluator (first-try parse + structure + grounding) as the fitness function.

The evaluator made a genuinely correct call in its first live run: it caught variant A embedding a single-use example into a reusable method — exactly what a bad method looks like — and scored it below baseline. No human told it that.

07 · the family

one principle, four scales

Three systems we studied deeply converged on the same rule from three directions. This project is that family's personal-memory member:

AlphaEvolve · DeepMind

evolves: algorithms
verifier: automated evaluator
peak: beat Strassen after 56 years; recovered 0.7% of Google's compute

SkillOpt · Microsoft

evolves: skill documents
verifier: held-out validation gate
peak: +24.8 pts inside agent loops; proved the −52.8 ungated cliff

jcode

evolves: its own source code
verifier: CI ratchets (budgets that only tighten)
peak: 654k-line harness that maintains itself

project-rag · this

evolves: memory, retrieval policy, abilities
verifier: held-out gate + owner approval
constraint the others don't have: a frozen 4B model on 4GB, fully offline

Generate freely. Verify ruthlessly. Never regress below baseline.
The same sentence describes all four systems — discovered independently, at four different scales.
